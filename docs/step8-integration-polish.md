## 3. 错误处理与用户体验优化

### 3.1 全局错误处理

```dart
// 在 ApiClient 中统一处理
_dio.interceptors.add(InterceptorsWrapper(
  onError: (error, handler) async {
    String message;
    switch (error.type) {
      case DioExceptionType.connectionTimeout:
      case DioExceptionType.sendTimeout:
      case DioExceptionType.receiveTimeout:
        message = "网络连接超时，请稍后重试";
        break;
      case DioExceptionType.connectionError:
        message = "网络连接失败，请检查网络设置";
        break;
      default:
        final statusCode = error.response?.statusCode;
        if (statusCode == 401) {
          // Token 过期处理 (已在 Step 2 实现)
        } else if (statusCode == 403) {
          message = "无权限执行此操作";
        } else if (statusCode == 404) {
          message = "请求的资源不存在";
        } else if (statusCode != null && statusCode >= 500) {
          message = "服务器异常，请稍后重试";
        } else {
          message = error.response?.data?['detail'] ?? "未知错误";
        }
    }
    // 通过全局 SnackBar 显示错误
    handler.next(error);
  },
));
```

### 3.2 防重复提交

```dart
// 所有提交按钮使用此模式
bool _isSubmitting = false;

Future<void> _handleSubmit() async {
  if (_isSubmitting) return;
  setState(() => _isSubmitting = true);
  try {
    await _doSubmit();
  } finally {
    if (mounted) setState(() => _isSubmitting = false);
  }
}

// 按钮状态
ElevatedButton(
  onPressed: _isSubmitting ? null : _handleSubmit,
  child: _isSubmitting
    ? SizedBox(width: 20, height: 20, child: CircularProgressIndicator())
    : Text("确认"),
)
```

### 3.3 输入验证优化

- 手机号: 实时验证格式，不合法时按钮灰显
- 体重: 限制小数点后最多两位
- 日期: 不允许选择未来日期
- 宠物名字: 去除首尾空格，不允许为空

### 3.4 页面状态管理

每个数据页面需要处理 4 种状态:
1. **Loading**: 首次加载数据中 → 显示骨架屏
2. **Success**: 数据加载完成 → 正常展示
3. **Empty**: 数据为空 → 友好的空状态提示
4. **Error**: 加载失败 → 错误提示 + 重试按钮

## 8. 执行方式

本步骤的执行方式与前面步骤不同，不是从零开发新功能，而是:

1. **逐一过测试清单**: 打开 APP，按照第 1 节的测试清单逐项操作
2. **记录问题**: 发现的 Bug 和 UI 不一致的地方记录下来
3. **批量修复**: 将问题分类提交给 AI agent 修复
4. **再次验证**: 修复后重新测试对应功能
5. **UI 截图审查**: 每个页面截图，对照第 2 节的设计规范检查

建议分多轮进行，每轮聚焦一个模块 (认证→档案→照片→健康→时间轴→推送)。
