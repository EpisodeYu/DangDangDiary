# DangDangDiary 全局规则

本文档是后续 agent 的统一入口说明。

目标只有两个：
- 让跨步骤实现保持一致
- 避免因为读到旧约定或局部示例而跑偏

如果某个步骤文档有更明确、且不与本文冲突的说明，可以按步骤文档执行。

## 1. 总体原则

- 以当前 Phase 1 为主，优先完成稳定、可运行、可验证的实现
- 优先保持跨文档一致性，不要因为局部示例代码而引入另一套规则
- 遇到会影响接口、数据结构、权限、用户流程的重要歧义时，先停下来询问

## 2. 网络与访问

- 真机开发阶段采用统一入口方案
- 手机端只访问一个入口地址，不直接访问 FastAPI 或 MinIO 内部地址
- 默认由 Nginx 提供统一入口
- 推荐路径约定:
  - `/api/...` -> 后端接口
  - `/media/...` -> 媒体访问路径

## 3. 配置与敏感信息

- 不要把真实密钥、固定密码、服务器私有地址写入文档示例或代码默认值
- 后端通过 `.env` 读取配置，`.env.example` 只保留占位符
- 前端使用统一配置入口管理 `base_url`

## 4. API 默认约定

- 接口字段默认使用 `snake_case`
- 列表接口默认使用 `page` + `page_size`
- 列表响应保留业务语义 key，例如 `pets`、`photos`、`weights`
- 时间类记录默认按“最新在前”排序
- 创建/更新成功默认返回最新完整对象
- 删除成功默认返回 `204 No Content`
- 列表为空时返回 `200 + 空数组`

## 5. 错误处理

- 业务错误默认使用结构化格式:
  - `code`
  - `message`
  - `details`
- 输入不合法默认归类为 `400`
- 无权限访问默认使用 `403`

## 6. 认证

- 使用 `access_token` + `refresh_token`
- 支持 `logout`
- `logout` 默认只作废当前设备的 `refresh_token`

## 7. 媒体与时间

- 若用户选择 `HEIC/HEIF`，默认由前端先转换为 `JPEG` 后上传
- 后端稳定支持 `JPG` / `PNG` / `WEBP`
- 列表接口默认返回可直接展示的缩略图 URL
- 查看大图时再单独请求原图 URL
- 时间戳统一按 `UTC` 存储
- 生日、拍摄日期、记录日期等仅表示日期的字段继续使用 `date`

## 8. 共享档案权限

- `member` 默认可以新增、编辑、删除普通记录
- `member` 不能删除宠物档案本身
- 角色信息只在确实有用的对象中返回，例如 `pet`

## 9. 产品与实现取向

- Phase 1 追求“功能稳定 + 体验干净”
- 主导航入口至少应有基础可用页面
- 危险操作默认二次确认
- 表单尽量提供合理默认值
- 高级交互保留真正有价值的部分，不强求一次做满

## 10. Agent 工作方式

- 每个 step 开始前，先读取相关文档并给出计划
- 当前 step 明确必需的新依赖可以引入，但要说明原因
- 与当前 step 无关的问题可以小范围顺手修复，但不要扩散
- 每个 step 完成后尽量做充分验证
- 如果实现导致文档明显过时，可在同一 step 内顺手同步更新相关文档
